## 1.1.0 - 2026-07-23

### 知识准备

- 将论文知识准备收敛为固定顺序的一次性流程：论文背景完成后再写术语，最后仅进行一轮可选外部补充。
- 论文内提取均衡读取最多 8,000 个字符，生成五项非空背景字段和 6–12 个正文可逐字定位的高价值术语后立即结束；核心与外部流分别限制为 16,000/8,000 个可见字符和 2,000,000/1,000,000 个响应字节，外部补充最多 3 个问题、3 次搜索和 3 个来源，不递归扩展或自动重试。
- 修复同一论文的并发文件写入和阶段所有权冲突。`complete / warning / error / skipped` 均为终态，侧栏进度不会再从 3/5 退回 2/5。
- 模型请求长时间不返回时会被取消并显式标记失败或警告；卡死检测不作为成功条件，且不影响已经开放的 Markdown 翻译。
- `_preparation.json`、背景、术语和来源统一走每篇论文的串行原子写入；索引新增 manifest 哈希，来源变化时确定性失效。
- 重启恢复会重新解析五项背景字段和完整术语行，并逐项核对正文原词、来源等级、置信度和时间；不再用标题标记或表格行数冒充完成。
- 修复旧 Markdown 请求晚返回后覆盖新版背景、来源或术语的竞态；每次写入都会在目录锁内重新核对当前来源哈希和准备记录。
- 索引恢复改为与当前 Markdown 重新生成的分块逐项比对，空分块、越界偏移和错误邻接会直接重建，不再被标为完成。
- 修复知识准备向旧版 ChatGPT Codex 后端发送不兼容的 `max_output_tokens` 所导致的 HTTP 400；同时按 `llm-for-zotero` 的真实旧版协议移除 `max_tool_calls`，不做失败后的换参数重试。
- 输出量、完整流大小和网页搜索次数改由本地流解析器硬性限制，越界即取消并显式报错；外部检索只保存能与 Responses URL citation 精确对应的 HTTPS 来源，无来源的模型背景不会进入翻译上下文。
- 术语只在一次核心分析中生成 6–12 项，不再在每次翻译后发起额外术语请求；重启时发现背景、术语或外部来源只写了一半，会把未完成阶段明确收口，不再重新执行整篇论文分析。
- 修复划词翻译重新读取上下文时误停当前后台任务的问题；只有任务调度器能把重启后遗留的 `running` 判为陈旧，普通刷新不会改写活跃阶段。
- 可选知识文件损坏不再阻断来源与索引已就绪的翻译。完成记录保留单调终态，独立完整性问题写入 `_preparation.json`，无效内容不会注入模型。
- 外部摘要与来源记录增加同次内容哈希，并采用失败回滚的双文件写入；不会留下无来源摘要或错误的完成状态。
- `_preparation.json` 新增独立的准备 schema、当前 attempt 和只追加的历史快照。普通重开不再重新执行失败任务；人工重试会创建新 attempt，并保留旧终态供排查。
- 新 attempt 会在每次背景、术语、来源和阶段写入前核对论文身份、Markdown 哈希及 attempt ID，拒绝旧请求晚返回后覆盖新结果。
- 核心重试从第一个失败阶段继续，保留已校验背景和人工术语；外部重试只重置配对的外部摘要与来源记录，任一写入失败都会回滚。
- 缺少全部 MinerU 缓存文件与“缓存只缺一部分”现在使用不同错误类型。侧栏不再展示绝对路径，也不会把损坏缓存误判成未配置 Token。

### Reader 与界面

- 侧栏新增协调样式的“打开目录”按钮，只打开当前论文经过身份校验的知识目录。
- Reader 浮层改为一个可缩放容器；拖动右下角时，原文与译文区域按同一比例一起变化，流式输出不会自行改尺寸。
- 修复相同文本在不同附件间串用任务、空白选区沿用旧任务，以及侧栏重绘重复启动知识准备的问题。
- 修复快速切换 Reader 条目时上下文未缓存、侧栏编辑后仍显示旧译文，以及无任务浮层误显示“正在翻译”的问题。
- 流式翻译期间编辑原文会取消旧请求并创建独立任务，旧译文不会写回新原文。
- 将译文流刷新与知识文件刷新拆开，并按附件、Markdown 哈希和刷新序号拒绝晚到的旧读取；连续选区时，旧任务结束事件也不会再截断新任务的流式显示。
- Reader 关闭或切离附件时只取消该附件的活动翻译；知识文件行名和状态提示补齐中英文界面文本。
- 保留公式中的上标字符、独立数字正文和有语义的连字符；只有经 MinerU Markdown 逐字证明时才合并跨行断词。
- 文件列表旁新增“停止准备 / 重新准备 / 重试外部补充”动态按钮。停止只取消当前论文的知识任务，来源与索引完成后的翻译保持可用。
- MinerU Markdown 尚未生成时，在文件列表下方显示紧凑说明、Token 管理入口和“重新检查”按钮，不增加单独帮助卡。

### 稳定性与设置

- 目标语言固定为中文，避免同一论文术语文件混入不同目标语言；设置页显示只读值。
- 补齐 Reader、侧栏、通知和设置面板的显式注销，启动失败时回滚已注册组件。
- 插件关闭时取消仍在运行的核心背景与外部补充请求；翻译完成后不再启动额外术语请求。
- Codex 登录文件刷新改为串行原子更新，避免与 Codex CLI 或 `llm-for-zotero` 同时刷新时覆盖新令牌。
- Codex 共享令牌刷新增加独立 30 秒上限和关闭取消；SSE 解析支持跨网络分块的 CRLF，并校验网页引用事件。
- 设置页连通性测试增加 30 秒取消和关闭清理；OAuth 刷新期间若 Codex CLI 已写入新令牌，本插件保留并使用 CLI 的新值，不会覆盖登录状态。
- 修复永久更新地址的 `update.json` 发布流程，并增加状态机、并发写入、选区清理、任务隔离和联动缩放测试。
- CI 和发布流程新增整套测试 60 秒上限、完整 Prettier 范围、XPI 内 manifest/私有内容及 `update.json` 校验；同版本同内容资产会跳过，不同内容会拒绝替换，固定更新清单也拒绝版本倒退。
- 新增 MinerU 缓存分类、旧状态迁移、attempt 归档、晚写屏障、外部配对重置、跨 Markdown 版本停止隔离、重试交接和阶段间隙停止测试；本版本共通过 98 项自动化测试。

## 1.0.0 - 2026-07-17

这是 `paper-translate-for-zotero` 的第一个正式版本。

### 翻译流程

- 从 `llm-for-zotero` 已生成并通过来源校验的 MinerU `full.md` 获取论文上下文，不读取 PDF，也不调用 MinerU。
- 论文来源和章节索引写完后立即开放翻译。背景分析、术语提取和网页检索转入后台，不再占用当前翻译的等待时间。
- 每次翻译只提交选中文本、相关正文片段、已积累的背景和术语，不重复发送全文。
- 新选区会取消当前翻译并替换原文；流式输出只显示译文。

### 选区与排版

- 过滤跨页选区中的 IEEE 版权号、授权提示、下载记录、限制提示和独立页码。
- 合并 PDF 正文的视觉断行，保留空段、项目符号和列表顺序。
- 要求模型按原文的段落和列表换行；流式结果会再次整理项目符号边界。
- Reader 浮层和侧栏中的原文框、译文框采用稳定的初始尺寸，输出过程中不再反复缩放。

### 论文知识文件

- 每篇论文使用 Zotero 父条目 key 建立独立目录。
- 新增 `_paper_source.json`、`_preparation.json`、`index.json`、`background.md`、`terminology.md` 和 `background-sources.json`。
- MinerU manifest 没有章节时，从 Markdown 的 `#` 到 `####` 标题重建章节和 UTF-16 偏移。
- 背景内容区分论文依据与外部补充。双语术语表记录原文写法、规范英文、中文译法、正文证据、来源等级和置信度。
- Markdown 哈希变化后重建索引并重新核对术语。正文仍有对应原词时，保留人工修改过的译法。
- 文件内容和进度状态使用原子写入；并发术语更新按论文串行落盘，避免互相覆盖。

### 背景检索

- 不再把 Crossref、Semantic Scholar 或任何固定网站设为翻译前置条件。
- 先根据论文内容生成检索问题，再由 Codex 进行可选网页搜索。
- 论文正文以及官方、标准资料优先；学术来源用于补充，社区页面只解释通用概念。
- 网络受限或没有结果时记录 `warning`，不阻断基于 Markdown 的翻译。

### 界面与设置

- Reader 侧栏显示论文标题、作者、年份、`MD` 标记和实际文件进度，不再输出整段背景或完整错误网址。
- 设置页名称固定为 `paper-translate-for-zotero`。
- 认证模式显示为不可编辑的 `Codex Auth`，删除界面中的 `(Legacy)` 字样和说明小字。
- 默认模型为 `gpt-5.4`，默认推理强度为 `medium`。
- 保留模型连通性测试。

### 独立性与错误处理

- 插件使用自己的 addon ID、偏好、DOM、CSS、提示词、任务和上下文目录，不读取或修改 `llm-for-zotero` 对话。
- Codex 请求使用 CLI 登录凭据和固定 Responses 地址，不启动 App Server，也不切换协议、模型或服务商。
- 来源冲突、缓存损坏、流式响应错误和路径越界会直接报错，不添加 PDF 回退或伪造成功结果。
- Zotero 条目进入回收站时保留上下文；只有父条目永久删除后才按路径和身份检查结果清理目录。

### 工程与发布

- 添加 AGPL-3.0-or-later 许可、NOTICE、上游复用说明、GitHub Actions CI 和标签发布流程。
- 发布前通过 36 项单元测试、TypeScript、ESLint、Prettier、生产构建和 XPI 内容检查。
