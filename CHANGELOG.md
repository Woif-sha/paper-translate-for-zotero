# 版本日志

本文件记录已经发布的版本。版本号遵循语义化版本规则。

## 1.0.0 - 2026-07-17

这是 `paper-translate-for-zotero` 的第一个正式版本。

### 翻译流程

- 从 `llm-for-zotero` 已生成并通过来源校验的 MinerU `full.md` 获取论文上下文，不读取 PDF，也不调用 MinerU。
- 论文来源和章节索引写完后立即开放翻译。背景分析、术语提取和网页检索转入后台，不再占用当前翻译的等待时间。
- 每次翻译只提交选中文本、相关正文片段、已积累的背景和术语，不重复发送全文。
- 新选区会取消当前翻译并替换原文；流式输出只显示译文。

### 选区与排版

- 过滤跨页选区中的 IEEE 版权号、授权提示、下载记录、限制提示和独立页码。
- 合并 PDF 正文的视觉断行，保留空段、项目符号和列表顺序。
- 要求模型按原文的段落和列表换行；流式结果会再次整理项目符号边界。
- Reader 浮层和侧栏中的原文框、译文框改为固定等高尺寸，输出过程中不再反复缩放。

### 论文知识文件

- 每篇论文使用 Zotero 父条目 key 建立独立目录。
- 新增 `_paper_source.json`、`_preparation.json`、`index.json`、`background.md`、`terminology.md` 和 `background-sources.json`。
- MinerU manifest 没有章节时，从 Markdown 的 `#` 到 `####` 标题重建章节和 UTF-16 偏移。
- 背景内容区分论文依据与外部补充。双语术语表记录原文写法、规范英文、中文译法、正文证据、来源等级和置信度。
- Markdown 哈希变化后重建索引并重新核对术语。正文仍有对应原词时，保留人工修改过的译法。
- 文件内容和进度状态使用原子写入；并发术语更新按论文串行落盘，避免互相覆盖。

### 背景检索

- 不再把 Crossref、Semantic Scholar 或任何固定网站设为翻译前置条件。
- 先根据论文内容生成检索问题，再由 Codex 进行可选网页搜索。
- 论文正文以及官方、标准资料优先；学术来源用于补充，社区页面只解释通用概念。
- 网络受限或没有结果时记录 `warning`，不阻断基于 Markdown 的翻译。

### 界面与设置

- Reader 侧栏显示论文标题、作者、年份、`MD` 标记和实际文件进度，不再输出整段背景或完整错误网址。
- 设置页名称固定为 `paper-translate-for-zotero`。
- 认证模式显示为不可编辑的 `Codex Auth`，删除界面中的 `(Legacy)` 字样和说明小字。
- 默认模型为 `gpt-5.4`，默认推理强度为 `medium`。
- 保留模型连通性测试。

### 独立性与错误处理

- 插件使用自己的 addon ID、偏好、DOM、CSS、提示词、任务和上下文目录，不读取或修改 `llm-for-zotero` 对话。
- Codex 请求使用 CLI 登录凭据和固定 Responses 地址，不启动 App Server，也不切换协议、模型或服务商。
- 来源冲突、缓存损坏、流式响应错误和路径越界会直接报错，不添加 PDF 回退或伪造成功结果。
- Zotero 条目进入回收站时保留上下文；只有父条目永久删除后才按路径和身份检查结果清理目录。

### 工程与发布

- 添加 AGPL-3.0-or-later 许可、NOTICE、上游复用说明、GitHub Actions CI 和标签发布流程。
- 发布前通过 36 项单元测试、TypeScript、ESLint、Prettier、生产构建和 XPI 内容检查。
